# 顔絵文字による感情スタイル制御ガイド

Irodori-TTS の VoiceDesign モデルは、`--caption` パラメータに顔絵文字を含めることで
音声の感情・スタイルを制御できます（"Emoji-driven Style Control"）。

## 仕組み

VoiceDesign モデル（`Aratako/Irodori-TTS-500M-v2-VoiceDesign`）は、
テキストとは別に「スタイルキャプション」を受け取るキャプションエンコーダを持ちます。
このキャプションに顔絵文字を入れると、その絵文字に対応した感情・雰囲気で発話を生成します。

```
--text   "合成したいテキスト"
--caption "😊 明るく元気な女性の声で話す"   ← ここに絵文字を入れる
```

## テキスト正規化の注意点

`text_normalization.py` で以下の記号は**変換・削除**されます：

| 入力 | 出力 | 備考 |
|------|------|------|
| `♥` | `♡` | 赤ハート→白ハートに変換 |
| `●` `◯` `〇` | `○` | 丸記号に統一 |
| `～` `〜` | `ー` | 長音符に変換 |
| `▼` `♀` `♂` | （削除） | 削除 |
| `；` `《》≪≫` `①②…` | （削除） | 削除 |

**Unicode 絵文字（😊 等）は正規化されず、そのまま保持されます。**

## 顔文字以外の絵文字

顔絵文字以外の絵文字もキャプションに含めることで音声スタイルに影響を与えられます。
**`--caption` に使用する場合**（VoiceDesign モデル）は顔絵文字と組み合わせると効果的です。

> **Note**: `--text` に絵文字を含める場合、Unicode 絵文字は `text_normalization.py` の
> NFKC 正規化をそのまま通過しますが、ただし `♥`（U+2665）→ `♡` への変換は適用されます。

## 基本的な使い方

```bash
uv run python infer.py \
  --hf-checkpoint Aratako/Irodori-TTS-500M-v2-VoiceDesign \
  --text "合成したいテキスト" \
  --caption "😊 明るく元気な女性の声で話す" \
  --no-ref \
  --output-wav outputs/sample.wav
```

## 絵文字と日本語説明の組み合わせ（推奨）

絵文字単体より「絵文字 + 日本語説明」の組み合わせが効果的です。

```bash
# 悲しみ例
--caption "😢 悲しみをこらえながら、静かに語りかけるような女性の声で"

# 明るさ例
--caption "😄 元気いっぱいで、笑顔が伝わるような若い女性の声で"

# クール例
--caption "😎 落ち着いた低めの声で、クールに淡々と話す"
```

## ローカル学習モデルへの適用（基本モデル + 参照音声）

LoRA 学習済みモデル（mamimi等）では VoiceDesign 機能は使えませんが、
参照音声＋テキストに絵文字を入れることで一定の感情表現が可能です。

```bash
uv run python infer.py \
  --checkpoint outputs/mamimi_lora/mamimi_best.safetensors \
  --text "今日は本当に楽しかったです！😊" \
  --ref-wav data/mamimi/wavs/seg_00338.wav \
  --output-wav outputs/sample.wav
```

> **Note**: テキスト中の絵文字は `llm-jp/llm-jp-3-150m` トークナイザで処理され、
> プロソディ（抑揚・感情）に影響を与える可能性があります。
