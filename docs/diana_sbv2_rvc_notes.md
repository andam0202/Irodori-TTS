## Style-Bert-VITS2 で Diana TTS モデルを作る場合

- **学習要件**: 2〜14秒の音声クリップ + 書き起こし → `data/diana_v2/wavs/`（195クリップ、1.5〜12秒、metadata.csv 付き）が**そのまま合致**
- metadata.csv → esd.list 形式への変換は数行のスクリプトで可能
- データ量 約10分は「動くが少なめ」の水準

### 合成データ（diana_v2 LoRA 自家生産）による増強の注意点

1. **順序が重要**: まず実データ195本のみで学習 → 品質評価 → 不足時のみ合成データ追加
   （最初から混ぜると問題発生時の原因切り分けが不可能になる）
2. 品質上限は LoRA の出力品質で頭打ち（蒸留）。発音ミス・韻律の癖も学習される
   → **人間による選別（良品のみ採用）が品質を左右する**
3. `infer.py` は silentcipher 透かしを埋め込む。合成データ比率は控えめに
   （実データと同量〜2倍程度まで）
4. 多様性確保のため参照音声・seed・キャプションを散らして生成する
5. 利点: テキストを自由に選べるため音素カバレッジを稼げる

## F5-TTS の NSFW 適性（2026-06-11 追記）

英語版 Diana 用に F5-TTS を導入した際の評価。**NSFW・非言語音声は F5-TTS の弱点**:

1. ベースモデルの学習データ（Emilia: ポッドキャスト・オーディオブック約10万時間）に
   喘ぎ・吐息等がほぼ含まれず、非言語発声は不安定（読み上げ化 or アーティファクト）
2. Irodori VoiceDesign のキャプション + 絵文字に相当するスタイル制御が無い
   （参照音声の音色を真似る間接制御のみ）
3. NSFW データで fine-tune は可能だが、書き起こしアライメントが難しく割に合わない

**結論**: NSFW/感情表現 = Irodori-TTS、英語のクリーン台詞 = F5-TTS と役割を分ける。

## RVC で Diana VC モデルを作る場合

- 書き起こし不要・音声10分程度で学習可能 → `data/diana_v2/wavs/` をそのまま流用できる
- リアルタイム変換（ボイチェン）にも対応
- 推奨構成: **TTS は Irodori / SBV2、VC は RVC** と役割を分ける
